## 概要
シンプルな操作でタスク管理ができるWebアプリです。 
タスクの追加・完了管理・削除に加え、期限設定とフィルター機能を実装しています。
ブラウザのローカルストレージを使用してデータを保存するため、ページをリロードしてもタスクが保持されます。

----

## 設計思想
本アプリでは「シンプルで迷わない操作」を重視しました。  
機能を最小限に絞ることで、ユーザーが直感的に使える設計にしています。
また、状態管理とUI更新を分離することで、機能追加や修正がしやすい構造にしました。

----

## 作成した背景
日々の学習や作業の中で、やるべきタスクを整理する重要性を感じていました。  
しかし、複雑なタスク管理ツールは操作が多く、継続して使うハードルが高いという課題がありました。
そこで、最小限の操作でタスク管理ができるシンプルなアプリを作成しました。

----

## 機能
* タスクの追加
* タスクの削除
* タスクの完了 / 未完了の切り替え
* フィルター機能（すべて / 未完了 / 完了）
* localStorageによるデータ永続化（リロード後も保持）
* 期限（締切日）の設定

----

## 使い方
1. 入力欄にタスクを入力
2. 「追加」ボタンをクリック
3. タスクをクリックすると完了 / 未完了が切り替わる
4. 「削除」ボタンでタスクを削除
5. フィルターで表示を切り替え

----

## 使用技術
* HTML
* CSS
* JavaScript（Vanilla JS）
* localStorage

----

## 工夫した点
* フィルター機能を実装し、状態に応じた表示切り替えをできる設計し、ユーザーがタスクを管理しやすいようにした。
* ユーザーがタスクを追加したときに、データが反映されいないと混乱しないよう、タスク追加時に「すべて」表示に戻るようにしました。
* CSSで全て、完了、未完了のうち選択したものに色を付けることでアクティブ状態を視覚的に分かりやすく表現しました。

----

## 設計・実装のポイント
- 配列とオブジェクトを使い、タスクの状態（完了 / 未完了）を管理できるようにしました。
- localStorageでデータを永続化することでリロード後もデータが消えないようにしました。
- DOM操作で動的に表示を更新するようにしました。
-　状態変更のたびに部分更新すると不整合が起きやすいため、render関数でUIを一括再描画する設計にしました。
- saveTasks関数とrender関数にわけることで、状態の更新とUIの描画を分離することができ、コードの可読性と拡張性を高めています。
- IDを用いてタスクを一意に管理し、削除処理の正確性を高めました。
- if (taskText.trim() === "") return;を使い、見た目上は入力されていても実質空文字になるケースを防ぎ、不要なデータ登録を防止しています。

----

## 苦労した点
フィルター機能において、「表示の状態」と「タスクの状態管理」を両立させることに苦労しました。 
当初は条件分岐が複雑になり、表示とデータの整合性が崩れる問題がありました。
そこで、表示に関する処理をrender関数に集約し、状態の変更はイベント処理側で行うように役割を分離しました。
その結果、コードの見通しが良くなり、バグも減らすことができました。
