# タスク管理アプリ（Todo App）

## アプリURL
https://omiu-de-ta.github.io/todo-app/

## アプリ画面


## 概要
シンプルな操作でタスク管理ができるWebアプリです。 
やるべきことを直感的に整理したいという目的で作成しました。

## 作成した背景
日々の学習や作業の中で、やるべきタスクを整理する重要性を感じていました。  
しかし、複雑なタスク管理ツールは操作が多く、継続して使うハードルが高いという課題がありました。
そこで、最小限の操作でタスク管理ができるシンプルなアプリを作成しました。


## 機能
* タスクの追加
* タスクの削除
* タスクの完了 / 未完了の切り替え
* フィルター機能（すべて / 未完了 / 完了）
* localStorageによるデータ永続化（リロード後も保持）


## 使い方
1. 入力欄にタスクを入力
2. 「追加」ボタンをクリック
3. タスクをクリックすると完了 / 未完了が切り替わる
4. 「削除」ボタンでタスクを削除
5. フィルターで表示を切り替え


## 使用技術
* HTML
* CSS
* JavaScript（Vanilla JS）
* localStorage


## 工夫した点
* フィルター機能を実装し、状態に応じた表示切り替えをできる設計にした
* ユーザーが混乱しないよう、タスク追加時に「すべて」表示に戻るようにした
* CSSで全て、完了、未完了のうち選択したものに色を付けることでアクティブ状態を視覚的に分かりやすく表現した

## 設計・実装のポイント
- 配列とオブジェクトを使い、タスクの状態（完了 / 未完了）を管理
- localStorageでデータを永続化することでリロード後もデータが消えないようにしました。
- DOM操作で動的に表示を更新すうるようにしました。
- saveTasks関数を作成し、データ保存処理を共通化しました。
- IDを用いてタスクを一意に管理し、削除処理の正確性を向上
- trim()を使用し、空白入力を防止を強化しました。
- render関数で画面を一括更新することで、シンプルな構造にしました


## 苦労した点
フィルター機能の実装において、表示条件の分岐とUIの状態管理の整合性を保つことに苦労しました。  
条件分岐を整理し、状態と表示を明確に分けることで解決しました。


## 学んだこと
- DOM操作・イベント管理の重要性
- UXを意識した設計の大切さ
- localStorageを使ったデータ永続化
